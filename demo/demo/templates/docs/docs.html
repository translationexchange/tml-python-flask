{% trs "Translation Markup Language" %}

{% trs "Translation Markup Language (TML) is used to identify the non-translatable or dynamic datain the labels. It provides a way to mark data and decoration tokensin the strings that need to be translated. There are different types of applications that can use TML - web, mobile and desktop. Some use HTML, others use Wiki-Like syntax for decorating the labels. TML aims at abstracting out the decoration mechanisms of the string used by the applications and instead provides its own simple, but powerful syntax. This allows for translation sharing across multiple applications." %}

{% tr %}Basics{% endtr %}

{% tr %}Flask Tml Client SDK provides translation function using template tags and filters. The most popular for translations are "tr" and "trs".{% endtr %} {% trs "The template tag can be called similar to trans of native Flask tag:" %} 
{­% tr bar=foo|filter, boo=baz|filter, description="description" %­}This is {{ bar }} and {{ boo }}.{­% endtr %­}
{% trs "You can skip context (tml description):" %} 
{­% tr bar=foo|filter, boo=baz|filter %­}This is {{ bar }} and {{ boo }}.{­% endtr %­}
{% trs "You can trim space character before and after the tag using `trim` option:" %} 
{% raw %}{% tr bar=foo|filter boo=baz|filter trimmed %}This is {{ bar }} and {{ boo }}.{% endtr %}{% endraw %}
{% trs "If you do not wish your translated string to be wrapped into a tml tag, use `nowrap` option:" %} 
{% raw %}{­% tr bar=foo|filter, boo=baz|filter options={'nowrap':True} %­}This is {{ bar }} and {{ boo }}.{­% endtr %­}{% endraw %}
{% trs "If you wish to store your translated string as context variable rather then print it, use `as` option::" %} 
{% raw %}{% tr bar=foo|filter, boo=baz|filter as foo_string %}This is {{ bar }} and {{ boo }}.{% endtr %}
{{ foo_string }}            {% endraw %}
{% trs "Tml in conjunction flask native i18n tags and filters provides a number of utilities:" %} 
{% raw %} {% trs "Hello world" %} {% endraw %}
{% raw %} {% tropts locale="ru" %}{% trs "Hello world" %} {% endtropts %}{% endraw %}
{% trs "To redefine generic tml options such as source or target locale use:" %} 
{% raw %} {% tropts source="navigation" %} ... {% endtropts %}{% endraw %}

  • {% trs "[bold: label] is the only required parameter." %}
  • {% trs "[bold: description] is an optional parameter, but should always be used if the label by itself is not sufficient enough to provide the meaning of the phrase." %}
  • {% trs "[bold: tokens] is an optional parameter that contains a hash (or a dictionary) of token values to be substituted in the label." %}
  • {% trs "[bold: options] provides a mechanism for passing additional directives to the translation engine." %}

{% trs "The context of a phrase is not mandatory, but it should be used in cases when the label alone is not sufficient enough to determine the meaning of the sentence being translated." %} {% trs "As a general rule, you should always provide context description to words, phrases and sentences that are only meaningfulin a specific context." %} {% trs "Tml uses label and description together to create a unique key for each phrase." %} {% trs "The description serves two purposes: it creates a unique key for each label and it also gives a hint to the translators for the context in which the label is used." %}

{% trs "For example, the following two phrases will be registered as two independent entries in a database even though the have the same label, but a different context. The user will have to translate each one of them separately as they will have different translated labels in other languages." %}

{% trs "example" %}
{­% trs "Invite", description="Link to invite your friends to join the site" %­}
{­% trs "Invite", description="An invitation you received from your friend" %­}
{% trs "Invite", description="Link to invite your friends to join the site" %}
{% trs "Invite", description="An invitation you received from your friend" %}

{% trs "It is important to provide the best possible description for each phrase from the start. Keep in mind that changing a description in the future, after it has already been translated, will register a new phrase in the database and invalidate all of its translations. On the other hand, labels that are complete sentences may not need a description as they are fully self-contained." %}

{% trs "Data Tokens" %}

{% trs "In many cases your tokens would be string objects that get substituted directly into the translated sentence." %}

{% trs "example" %}
{­% tr user=users.michael %­} Hello {user} {­% endtr %­}
{% tr user=users.michael %} Hello {user} {% endtr %}

{% trs "Translations can be nested." %}

{% trs "example" %}
{­% tr city="Los Angeles"|trs %­} Welcome to {­city­} {­% endtr %­}
{% tr city="Los Angeles"|trs %} Welcome to {city} {% endtr %}
{% tr city="Moscow"|trs %} Welcome to {city} {% endtr %}

{% trs "But make sure that you don't take translations out of context." %}

{% trs "example" %}
{­% tr link="", registration="registration page"|trs %­}Please visit our [link: {registration}] to join our site. {­% endtr %­}
{% tr link="", registration="registration page"|trs %}Please visit our [link: {registration}] to join our site. {% endtr %}

{% trs 'The problem the above example, is that the "registration page" link text would be translated differently based on the context where it appears.' %} {% trs "You must keep the two parts together to make sure the translations are accurate. You will later see how you can use decoration tokens to fix the above problem." %}

{% tr %}You can also get the substitution value by invoking a method or access hash key of the object using default jinja2 template syntax.{% endtr %}

{% tr %}example{% endtr %}
{­% tr user=[users.michael, ':name'] %­} Dear {user} {­% endtr %­}
{% tr user=[users.michael, ':name'] %} Dear {user} {% endtr %}

{% trs "Method Tokens" %}

{% trs "Method tokens allow you to invoke methods on the object you are passing to get the substitution value." %}

{% trs "example" %}
{­% tr user=users.michael %­}Hello {user.name}, you are a {user.gender}{­% endtr %­}
{% tr user=users.michael %}Hello {user.name}, you are a {user.gender}{% endtr %}

{% trs "Piped Tokens" %}

{% trs "Piped tokens work in conjunction context rules and allow you to provide substitution values based on the object values." %}

{% trs "example" %}
{­% tr count=1 %­}You have {count||one:message,other:messages}{­% endtr %­}
{% tr count=1 %}You have {count||one:message,other:messages}{% endtr %}
{% tr count=2 %}You have {count||one:message,other:messages}{% endtr %}
{% tr count=10 %}You have {count||one:message,other:messages}{% endtr %}

{% trs 'Double pipe "||" means that the value would be displayed, followed by the word that depends on the value. In this case, if the count value meets the criteria for the rule "one", then it will display the word set to the rule. For all other cases it would display the "other" value.' %}

{% trs 'Since the sequence of parameters is mapped to the sequence of rules, you can omit naming the parameters.' %}

{% trs "example" %}
{­% tr count=2 %­}You have {count||message,messages}{­% endtr %­}
{% tr count=1 %}You have {count||message,messages}{% endtr %}
{% tr count=2 %}You have {count||message,messages}{% endtr %}
{% tr count=10 %}You have {count||message,messages}{% endtr %}

{% trs "Since the library comes default pluralizers, you don't even need to provide the plural form. It will be automagically generated for you." %}

{% trs "example" %}
{­% tr count=2 %­}You have {count||message}{­% endtr %­}
{% tr count=1 %}You have {count||message}{% endtr %}
{% tr count=2 %}You have {count||message}{% endtr %}
{% tr count=10 %}You have {count||message}{% endtr %}

{% trs 'The same exact concept applies to other token types and context rules.' %}

{% trs "example" %}
{­% tr user=users.michael %­}{user} updated {user|his,her} profile.{­% endtr %­}
{% tr user=users.michael %}{user} updated {user|his,her} profile.{% endtr %}

{% trs 'Single pipe "|" means to not display the actual token value, but display the value that follows based on the context rules.' %}

{% trs "The context rules are specific for each language. But you don't have to worry about it, translators will provide the rules and the values." %}

{% trs "example" %}
{­% tr user=users.michael, options={'locale':'ru'} %­}{user||родился,родилась,родился/лась} в Ленинграде.{­% endtr %­}
{% tr user=users.michael, options={'locale':'ru'} %}{user||родился,родилась,родился/лась} в Ленинграде.{% endtr %}
{% tr user=users.anna, options={'locale':'ru'} %}{user||родился,родилась,родился/лась} в Ленинграде.{% endtr %}

{% trs "Implied Tokens" %}

{% trs "Implied tokens use a single pipe since the value does not need to be displayed." %}

{% trs "example" %}
{­% tr user=users.michael %­}{user|male:He,female:She,other:It} likes this movie.{­% endtr %­}
{% tr user=users.michael %}{user|male:He,female:She,other:It} likes this movie.{% endtr %}

{% trs "Similar to the previous examples, you don't have to provide the named parameter values." %}

{% trs "example" %}
{­% tr user=users.michael %­}{user|He,She,It} likes this movie.{­% endtr %­}
{% tr user=users.michael %}{user|He,She,It} likes this movie.{% endtr %}
